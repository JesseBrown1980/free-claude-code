## Provider possibility versus live result

Keep these states separate:

```text
provider/model appears in public catalog
proxy implementation supports provider class
local configuration/key is present
health/auth request succeeds
real inference call succeeds
Asolaria route emits a verified receipt
```

A catalog row is real capability/possibility evidence. It is not a live completion receipt.
A missing key is a configuration boundary, not proof that the provider adapter is fake.

## Trilateral procedure

- origin seat records the exact provider, model, route, status, latency, and receipt without secrets;
- attack-verify seat tests protocol/error handling with its own lawful credentials or local model;
- third seat runs public tests and, optionally, a separate own-account/local-model call;
- CI executes credential-free tests and mocks against the immutable commit.

No seat shares API keys merely to satisfy the triangle.

## Free and zero-cost scope

“Free” must say which mechanism applies:

```text
provider free tier
provider free model
local LM Studio / llama.cpp
subscription-backed ordinary use
cached/deterministic local interception
operator-reported Asolaria free-agent lane
```

Do not collapse those into one universal zero-cost guarantee.

## De-deflate / no-inflate

- adapter source plus tests are real even before a key is configured;
- a provider listing is not a successful live call;
- one successful model call is not proof every catalog model works;
- rate-limit/account terms remain provider-specific and current;
- local proxy capability is not automatically a live Asolaria fabric route;
- an Asolaria receipt does not transfer upstream project ownership or endorsement.

## Security and authority

Credentials remain local and uncommitted. Third-seat verification cannot receive operator secrets,
change provider accounts, launch uncontrolled background agents, or authorize external publication.

## Merge rule

Merge adapter fixes, mocks, error handling, local-model support, secret scans, and scoped receipts
when checks pass. Hold credentials, live-account assumptions, provider-policy claims, autonomous
execution expansion, or a catalog-to-live promotion without its own call receipt.
